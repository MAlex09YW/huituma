# 回图码（私密加密版）

一个完全在浏览器本地运行的私密二维码图片工具。图片使用 AES-GCM 在本机加密，GitHub 只保存加密文件；解密钥匙只存在于二维码中。

## 第一次发布网站

1. 在 GitHub 创建一个公开仓库，建议命名为 `huituma`。
2. 将本项目的 `docs` 文件夹和 `README.md` 上传到仓库根目录。
3. 打开仓库的 **Settings → Pages**。
4. 在 **Build and deployment** 中，将 **Source** 设为 **Deploy from a branch**。
5. Branch 选择 **main**，文件夹选择 **/docs**，然后点击 **Save**。
6. 等待 GitHub 显示网站地址，例如 `https://你的用户名.github.io/huituma/`。

## 制作并发布私密图片

1. 打开 GitHub Pages 网站。
2. 选择图片、二维码位置和大小，点击“生成加密二维码图片”。
3. 分别下载“含二维码图片”和随机编号的 `.bin` 加密文件。
4. 含二维码的 PNG 只用于发送、展示或打印，不要上传到公开仓库。
5. 将 `.bin` 文件上传到仓库的 `docs/vault` 文件夹，保持文件名不变。
6. 等待 GitHub 自动重新发布，再用另一台设备扫描二维码测试。

## 隐私说明

- 原图和含二维码的 PNG 都只在本机浏览器中处理。
- GitHub 仓库里只能上传 `.bin` 加密文件。
- 解密钥匙位于二维码网址的 `#` 后面，不会发送给 GitHub。
- 得到二维码、二维码照片或完整扫码链接的人都能查看图片；二维码应当像钥匙一样保管。
- 需要撤销访问时，从 `docs/vault` 删除对应 `.bin` 文件即可。
