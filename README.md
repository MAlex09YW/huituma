# 私密图片发布器（GitHub Pages 免费版）

用途：先生成一个独立二维码，手动把二维码放进图片，再将“已经带二维码的最终图片”加密发布。扫描二维码后，浏览器在纯白页面中只显示这张最终图片。

## 更新已经发布的网站

将本项目的整个 `docs` 文件夹上传到 GitHub 仓库根目录，覆盖旧文件并提交。保留 `docs/vault` 里已经发布的 `.bin` 文件，不要删除它们。

GitHub Pages 仍然使用：

- Branch：`main`
- Folder：`/docs`

## 第一次设置自动发布

1. 打开网站，点击右上角“发布设置”。
2. 扫码网址填写 `https://malex09yw.github.io/huituma/`。
3. GitHub 用户名填 `MAlex09YW`，仓库名填 `huituma`，分支填 `main`，目录填 `docs/vault`。
4. 在 GitHub 创建 Fine-grained personal access token：Repository access 只选择 `huituma`；Repository permissions 只把 `Contents` 设为 `Read and write`。
5. 把 token 填进设置，点击“测试连接”，成功后保存。
6. 只有在自己的私人设备上，才勾选“记住 token”。

## 以后每张图片的流程

1. 点击“生成并下载二维码”。
2. 用任意图片编辑软件，手动把刚下载的二维码放进图片，并导出最终图片。
3. 回到网站，选择这张已经带二维码的最终图片。
4. 点击“加密并自动发布”。
5. 网站提示发布完成后，用手机扫描图片里的二维码测试。

不要在同一张成品图制作过程中重新生成二维码；重新生成后，新旧二维码属于不同项目。

## 扫码网址可以改成什么

- 免费项目网址：`https://用户名.github.io/仓库名/`
- 免费个人主页网址：`https://用户名.github.io/`（仓库必须命名为 `用户名.github.io`）
- 自己拥有并已经绑定到 GitHub Pages 的根域名：`https://example.com/`
- 自己拥有并已经绑定的子域名：`https://www.example.com/`、`https://photo.example.com/`

不能填写别人拥有的域名，也不能只改输入框就获得一个新域名。自定义域名需要先购买或拥有该域名，再在 GitHub Pages 和域名 DNS 中完成绑定。正式二维码必须使用 HTTPS。

域名一旦写进二维码，之后再修改网址，旧二维码不会自动改变。因此应先确定网址，再批量制作二维码。

## 隐私和撤销

- 最终图片在浏览器本机使用 AES-GCM 加密。
- GitHub 公开仓库只保存随机文件名的 `.bin` 加密数据。
- 解密钥匙只在二维码链接的 `#` 后面；浏览器请求服务器时不会把这部分发送给 GitHub。
- 拿到二维码、二维码照片或完整扫码链接的人都能看图，所以二维码本身就是钥匙。
- 要让扫码页面失效，可从 `docs/vault` 删除对应的 `.bin` 文件；但别人已经保存的图片无法收回，公开仓库的历史提交也不能当作彻底擦除机制。
- GitHub token 不会写入二维码或加密文件；它只用于把 `.bin` 上传到你指定的仓库。

## 手动发布备用方案

如果自动发布失败，页面会出现“下载 .bin 手动上传”。下载后，将该文件原名上传到仓库的 `docs/vault` 文件夹，等待 GitHub Pages 更新即可。
